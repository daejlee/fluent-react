# 10. 리액트 대체제

## 10.1 뷰

- 앵귤러JS 프로젝트에 참여했던 `에반 유`가 개발한 프레임워크로, 앵귤러의 좋은 부분을 가져와 더 가볍고 유지보수가 용이하며 특정 방식에 얽매이지 않음

**과하게 도드라지지 않는 반응형 시스템**

- 컴포넌트 상태가 반응형 자바스크립트 객체로 구성되어, 이를 수정하면 뷰가 업데이트 됨

- 뷰의 반응형 모델은 객체 속성의 읽기, 쓰기를 가로챔

- **뷰 2**: 브라우저 지원 제한으로 게터와 세터만 사용 -> **뷰 3**: 반응형 객체에 프록시를 사용하고 참조에 게터와 세터 사용

```
// 객체를 인자로 받아 객체의 프록시를 반환해 get, set작업 가로챔
function reactive(obj) {
	return new Proxy(obj, {
		get(target, key) {
			track(target, key)
			return target[key]
		},
		set(target, key, value) {
			target[key] = value
			trigger(target, key)
		}
	})
}

// 객체 내에서 값을 캡슐화 하고 해당 값에 대해 반응형 get, set 작업 제공
function ref(value) {
	const refObject = {
		get value() {
			track(refObject, 'value')
			return value
		},
		set value(newValue) {
			value = newValue
			trigger(refObject, 'value')
		}
	}
	return refObject
}
```

### 10.1.1 시그널

- 뷰의 컴포지션 API에서 제공하는 refs와 유사한 반응형 특성

- 값에 접근할 때 의존성을 체크 / 값이 변경될 때 사이드이펙트를 일으키는 값 컨테이너

- 세분화된 구독을 통해 업데이트를 수행하는 렌더링 모델과 함께 자주 논의됨

- 가상 dom을 사용하는 뷰는 유사한 최적화를 달성하기 위해 컴파일러 사용하고 있으나, 솔리드에서 영감을 받아 가상 dom에 의존하지 않고 뷰의 내장된 반응형 시스템을 많이 활용하는 새로운 컴파일 전략(베이퍼 모드) 모색 중

### 10.1.2 단순성

- HTML 파일의 스크립트 태그에 뷰 라이브러리를 포함하고 뷰 컴포넌트를 작성하면 되기 때문에 시작하기에 단순함

- 스캐폴딩을 위한 CLI 도구 제공

- 강력한 반응형 시스템, 템플릿 기반 구문, 잘 구조화된 컴포넌트 모델

### 10.2.1 변경 감지

- `zone.js`라는 라이브러리를 사용해 컴포넌트의 뷰에 변경 사항이 있는지 확인하는 변경 감지기가 있음

- 컴포넌트를 위에서 아래로 살펴보며 변경 사항을 찾음

- **변경 감지 매커니즘**을 주기적으로 실행해 데이터 모델의 변경 사항이 애플리케이션 뷰에 반영됨

- 성능이 뛰어나지만 애플리케이션에서 자주 실행하면 속도가 느려짐

- 조건부 요소 렌더링을 위한 *ngIf, 목록 렌더링을 위한 *ngFor 등 강력한 지시자와 구조체 제공 -> 직접 DOM 조작

### 10.3 스벨트

- 선언적 컴포넌트를 명령어 코드로 변환하는 컴파일러를 사용해 DOM 업데이트 => 적은 코드로 반응형 웹 애플리케이션 작성 가능

```
<script>
let count = 0;
function increment() {
  count += 1;
}
</script>
<div>{count}</div>
<button on:click={increment}>
  클릭하세요
</button>
```

- 스벨트에서는 반응형 동작이 자동으로 이루어지기에 API를 활용해 dom 업데이트 할 필요 없이 변수에 할당하기만 하면 나머지는 스벨트가 알아서 처리

- 가상dom 비교 및 패키지 단계가 없기 때문에 런타임 성능이 더 빨라짐.

- 동적 컴포넌트 타입과 같이 가상 dom 기반 프레임워크에서 제공하는 일부 동적 기능은 표현이 더 번거롭거나 장황함.

- $ 접두사를 가진 특별한 함수/키워드

**런타임 반응성**

- 컴파일 타임 반응성 사용 => $(레이블) 사용하는 코드를 설정해 의존성이 변경될 때 자동으로 다시 실행되도록 하는 경우, 의존성은 컴파일할 때 정해짐

### 10.4 솔리드

- 선언적 자바스크립트 라이브러리로, 컴포넌트 모델 기반은 리액트와 유사하지만, 반응형 특성에 기반하는 점이 다름

- 가상 dom 사용 대신 세분화된 반응형 시스템을 사용해 의존성을 자동으로 추적하고 dom을 직접 업데이트함

```
function Component() {
  const [count, setCount] = createSignal(0);

  return (
    <>
      <div>{count()}</div>
      <button onClick={() => setCount(count() + 1)}>증가</button>
    </>
  );
}
```

- 리액트라면 component 함수 전체가 계속 재렌더링 되었겠지만, 솔리드에서는 count를 구독하는 div만 업데이트 됨
