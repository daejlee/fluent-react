# 9장. 리액트 서버 컴포넌트 (RSC)

RSC는 서버에서 렌더링되는 멀티 페이지 애플리케이션과 클라이언트에서 렌더링되는 단일 페이지 애플리케이션의 장점을 결합해, 성능과 유지보수 편의성을 저해하지 않고 매끄러운 사용자 경험을 제공한다.

RSC는 서버에서 "실행"되는 새로운 종류의 컴포넌트다.
> 실제 "실행"은 RSC, 클라이언트 컴포넌트 동일하게 서버에서 "실행"된다.
> 실행의 의미는 뒤에 다시 나온다.

---

## 9.1 장점

- 계산 능력을 제어할 수 있는 서버에서만 실행 → 성능 예측이 용이해진다.
- 안전한 서버에서 실행되므로, 민감 정보 유출 걱정이 없다.
- 비동기로 동작할 수 있다.

---

## 9.2 서버 렌더링

서버에서 컴포넌트를 렌더링 후 리액트 엘리먼트 트리 생성 → 서버 렌더러는 리액트 엘리먼트 트리를 마크업으로 변환.

- 첫 번째 단계: RSC 렌더러
- 두 번째 단계: 서버 렌더러

### 렌더링 흐름

1. 서버에서 JSX 트리를 엘리먼트 트리로 변환
   ```jsx
   <div>
     <h1>안녕</h1>
     <p>나는 리액트가 좋아요!</p>
   </div>
   ```
   변환된 자바스크립트 객체:
   ```js
   {
     type: 'div',
     props: {
       children: [
         {
           type: 'h1',
           props: { children: '안녕' }
         },
         {
           type: 'p',
           props: { children: '나는 리액트가 좋아요!' }
         }
       ]
     }
   }
   ```
2. 서버는 이 엘리먼트 트리를 문자열이나 스트림으로 직렬화
3. 직렬화된 JSON을 클라이언트로 전송
4. 클라이언트 측 리액트가 문자열을 파싱해 렌더링

### 9.2.1 내부구조

- **문자열 타입**이라면 내장된 DOM 엘리먼트이므로 그대로 반환.
  - 다만, 자식으로 동시성 리액트 컴포넌트가 포함될 수도 있으므로 **재귀적으로 props에 대해 함수를 호출**해 처리한다.
  - → 자식 중에 또 다른 컴포넌트(함수)가 있을 수 있으니, 트리 전체를 다 처리할 때까지 계속 내려가며 실행한다는 의미.

- **함수 타입**은 `await`를 사용해 비동기 작업을 기다린 후, 데이터 의존성이 모두 해결된 엘리먼트 트리를 생성할 수 있다.

### 직렬화

직렬화가 중요한 이유:
1. 서버가 즉시 표시할 수 있는 HTML을 전송 가능
2. 성능 예측 가능성 확보
3. 클라이언트 측 하이드레이션 프로세스를 용이하게 한다
   - 직렬화된 트리(RSC Payload)가 "서버가 무엇을 렌더했는가"의 청사진 역할을 함
   - 클라이언트 React가 DOM을 처음부터 재계산하지 않고, 서버 결과 위에 이벤트 핸들러만 붙이면 됨

### 페이지 탐색

- 하드 네비게이션: 전체 페이지 새로고침
- 소프트 네비게이션: RSC Payload만 새로 받아 부분 업데이트 (SPA 방식)

### 9.2.2 업데이트

카운터 예시로 서버 컴포넌트와 클라이언트 컴포넌트의 업데이트 동작 비교.

### 9.2.3 주의할 점

- 서버 컴포넌트는 서버에서 실행되며 리액트 엘리먼트를 나타내는 객체를 출력한다.
- 클라이언트 컴포넌트도 서버에서 실행되며 리액트 엘리먼트를 나타내는 객체를 출력한다.
- 서버에서는 모든 리액트 엘리먼트를 표현하는 커다란 객체가 만들어진다.
- 이 객체를 문자열로 변환해 클라이언트로 전송한다.
- **이 시점부터 서버 컴포넌트는 실행되지 않는다.**
- **이 시점부터 클라이언트 컴포넌트는 클라이언트에서만 실행된다.**

#### "실행"이란 무엇인가?

props와 함께 호출되어 다음과 같은 평범한 자바스크립트 객체로 표현되는 리액트 엘리먼트를 반환하는 것.

```js
{
  type: 'div',
  props: { children: [...] },
  ...
}
```

---

## 9.3 서버 컴포넌트 규칙

1. **직렬화 가능성**이 가장 중요하다. — props는 직렬화 가능해야 함 (함수, 클래스 인스턴스 등 불가)
2. **부작용이 있는 훅 금지** — `useState`, `useEffect` 등 사용 불가
3. **상태는 동일하지 않다** — 서버 컴포넌트는 상태를 가지지 않음
4. **클라이언트 컴포넌트는 서버 컴포넌트를 import할 수 없다** — children으로 전달은 가능
5. **클라이언트 컴포넌트는 나쁘지 않다** — 인터랙션이 필요한 곳에 적절히 사용

---

## 9.4 서버 액션

서버에서 실행되는 함수를 클라이언트에서 직접 호출할 수 있는 메커니즘.
`'use server'` 지시어로 선언하며, 폼 제출이나 데이터 뮤테이션에 활용.
